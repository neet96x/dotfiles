# Skill: frontend-design

1. **เลี่ยง AI-slop**: อย่าใช้ของโหลๆ — font กลางๆ (Inter/Roboto/Arial), purple gradient บนพื้นขาว/ดำ, layout ตายตัวซ้ำๆ
2. **เลือกคอนเซ็ปต์ให้เข้ากับงาน**: dashboard ≠ editorial ≠ fintech ≠ portfolio — palette, typography, spacing ต้องสื่อบุคลิกของงานนั้น
3. ถ้าโจทย์เปิดกว้าง: **เสนอ 3-4 ทิศทาง** (สี/ฟอนต์/อารมณ์ พร้อมเหตุผลสั้นๆ) ให้ผู้ใช้เลือกก่อนลงมือ — อย่าเดาแล้วทำเลย
4. ดีเทลที่ทำให้ดูพรีเมียม: type scale ชัด, spacing สม่ำเสมอ, micro-interaction/transition, hover/focus state, empty/loading state
5. **Responsive** (mobile-first) + **contrast ผ่าน WCAG AA** (อ่านง่าย เข้าถึงได้)
6. ใช้ font/สีที่เจาะจง (ระบุ hex/ชื่อ font จริง) ไม่ลอยๆ

จับคู่กับ skill ของ frontend framework ที่ใช้ (เช่น nextjs-prisma) เมื่อต้อง implement
