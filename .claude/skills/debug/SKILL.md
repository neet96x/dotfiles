# Skill: debug

อย่าเดาแล้วแก้มั่ว — ไล่ตามนี้:

1. **Reproduce**: ทำให้บั๊กเกิดซ้ำได้ก่อน หา input/ขั้นตอนที่เล็กที่สุดที่ทำให้พัง (minimal repro)
2. **อ่าน error เต็มๆ**: stack trace อ่านจากล่างขึ้นบน หาบรรทัดในโค้ดเราที่ใกล้จุดพังสุด อ่าน error message ตามตัวอักษร
3. **ตั้งสมมติฐาน**: คิดว่าอะไรน่าจะเป็นสาเหตุ (1-2 ข้อ)
4. **พิสูจน์**: ใส่ print/log หรือรันเฉพาะส่วน เพื่อยืนยัน/ตัดสมมติฐาน — ดูค่าจริง ไม่ใช่ค่าที่คิดว่าเป็น
5. **แก้ที่ราก** (root cause) ไม่ใช่กลบอาการ
6. **กันซ้ำ**: เพิ่มเทสต์ที่ reproduce บั๊กนี้ (ดู skill write-tests)
7. **ยืนยัน**: รันใหม่ว่าหายจริง และไม่พังที่อื่น

ถ้าติดนาน: ตัดปัญหาครึ่งต่อครึ่ง (bisect) — comment โค้ดออกทีละส่วนจนเหลือจุดที่พัง
