# Skill: docs-write

1. คิดถึง**คนอ่าน**ก่อน: ผู้ใช้ทั่วไป หรือ dev — เขียนให้ตรงระดับนั้น
2. **README** โครงที่ดี: โปรเจกต์ทำอะไร+ทำไม → ติดตั้ง → วิธีใช้พร้อมตัวอย่างรันได้จริง → config → (จุดสำคัญอื่น)
3. **Docstring**: เฉพาะ public function/class — บอก args, returns, raises, ตัวอย่างถ้าซับซ้อน; ใช้รูปแบบที่โปรเจกต์ใช้อยู่ (Google/NumPy/JSDoc)
4. **คอมเมนต์**: อธิบาย "ทำไม" (เหตุผล/ข้อควรระวัง) ไม่ใช่ "ทำอะไร" (ซึ่งโค้ดบอกอยู่แล้ว) — อย่าคอมเมนต์สิ่งที่ชัดเจนอยู่แล้ว
5. ให้ docs **ตรงกับโค้ดจริง** — อ่านโค้ดก่อนเขียน อย่าแต่ง; ตัวอย่างต้องรันได้
6. กระชับ ตรงประเด็น มี code block ที่ก๊อปไปใช้ได้เลย
