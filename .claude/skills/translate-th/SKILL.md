# Skill: translate-th

1. แปลให้**เป็นธรรมชาติ** ตามความหมาย ไม่ใช่แปลคำต่อคำ
2. **คงไว้เป็นต้นฉบับ**: code, คำสั่ง, path, ชื่อเฉพาะ (brand/library/product), ศัพท์เทคนิคที่นิยมใช้ทับศัพท์ (เช่น API, deploy, commit, framework)
3. รักษา formatting เดิม: bullet, code block, ตาราง, ขึ้นบรรทัด
4. เอกสารเทคนิค: ศัพท์เฉพาะคงอังกฤษไว้ แล้วใส่คำอธิบายไทยในวงเล็บครั้งแรกถ้าจำเป็น
5. ถ้าประโยคกำกวมหลายความหมาย → เลือกที่เข้ากับบริบทโค้ด/งานมากสุด ไม่ต้องแปลตรงตัวจนงง

ตอบเฉพาะคำแปล เว้นแต่ผู้ใช้ขอคำอธิบายเพิ่ม
