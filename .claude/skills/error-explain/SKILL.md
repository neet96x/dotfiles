# Skill: error-explain

1. อ่าน error **ทั้งก้อนตามตัวอักษร** — stack trace อ่านจากล่างขึ้นบน หาบรรทัดในโค้ดเราที่ใกล้จุดพังสุด
2. ระบุ **ชนิด error** (เช่น `KeyError`, `NullPointerException`, `ECONNREFUSED`) และ**จุดที่เกิด**
3. อธิบายเป็นไทยง่ายๆ ว่า error นี้**แปลว่าอะไร**จริงๆ (ไม่ใช่แค่อ่านซ้ำ)
4. บอก **สาเหตุที่พบบ่อย** ของ error แบบนี้ (1-3 ข้อ เรียงจากน่าจะใช่สุด)
5. ให้ **ขั้นตอนแก้ที่ทำได้จริง** + วิธีเช็คว่าใช่สาเหตุไหน
6. ถ้าต้องไล่ลึก/reproduce → ใช้ skill debug ต่อ

ถ้าข้อมูลไม่พอ (ไม่มี stack/บริบท) บอกว่าต้องดูอะไรเพิ่ม
