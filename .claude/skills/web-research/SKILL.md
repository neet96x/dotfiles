# Skill: web-research

1. `web_search` ด้วยคำค้นที่เจาะจง (ภาษาอังกฤษมักได้ผลดีกว่าสำหรับเรื่องเทคนิค)
2. snippet มักเป็นแค่คำโปรย **ไม่มีคำตอบจริง** → ถ้าคำตอบ (ตัวเลข/ข้อเท็จจริง) ยังไม่ครบ ให้ `web_fetch` หน้าที่น่าเชื่อถือที่สุด 1-2 หน้ามาอ่าน
3. **เช็คอย่างน้อย 2 แหล่ง** สำหรับข้อเท็จจริงสำคัญ ถ้าขัดกันให้บอก
4. ตอบด้วย**ข้อมูลจริง** (ตัวเลข/ข้อสรุป) + อ้างอิง URL ที่มา ไม่ใช่แค่โยนลิงก์ให้ไปอ่านเอง
5. บอกวันที่ของข้อมูลถ้าเกี่ยวกับเวลา (ราคา/ข่าว) และบอกตรงๆ ถ้าไม่แน่ใจหรือหาไม่เจอ

ระวัง: หน้าที่เป็น JavaScript ล้วน (ticker ราคาสด) อาจ fetch ไม่ได้ตัวเลข — ลองหลายแหล่งหรือบอกข้อจำกัด
